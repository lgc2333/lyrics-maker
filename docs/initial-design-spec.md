# 歌词打轴软件需求文档

## 1. 项目概述

一款**Web 端歌词打轴软件**，提供类似 osu! editor 的 Timing 面板与 Aegisub 的字幕编辑体验。用户导入音频 → 建立 BPM/拍号时间轴 → 在网格上逐句/逐词打轴 → 导出为 LRC / ASS（SRT）/ JSON 等多种格式。

## 2. 技术选型

| 层次          | 技术                                                             |
| ------------- | ---------------------------------------------------------------- |
| 框架          | Vite + Vue 3 + TypeScript                                        |
| 自动导入      | unplugin-auto-import + unplugin-vue-components + vue-router/vite |
| UI 库         | DaisyUI v5 + Tailwind CSS v4                                     |
| 图标          | @iconify/vue (material-symbols)                                  |
| BPM 检测      | @libraz/libsonare (WASM)                                         |
| 音频处理      | 浏览器 Web Audio API；支持所有浏览器可解码格式                   |
| 时间轴引擎    | 待定                                                             |
| 节拍器        | 待定（需与音频实时同步并支持即时停止，跟随音频播放与停止）       |
| 频谱/波形     | 待定（支持波形/频谱切换，频率缩放可调）                          |
| 歌词解析/导出 | 待定（支持 LRC、ASS/SRT、JSON，模块可扩展）                      |
| 包管理        | pnpm                                                             |

> 所有标“待定”的技术选型请在实施前与产品方讨论确认。

## 3. 核心功能与工作流

**所有功能实现前，必须先为“撤销/重做”与“多语言”建立基础框架。**

### 3.1 新建打轴工作流

1. 导入音频文件（支持浏览器可解码的所有格式）。
2. 通过**时轴模式**建立 BPM/拍号时间轴：
   - 支持 TAP 测 BPM（最大保留最近 128 次敲击，间隔超过 1 秒则自动清空）。
   - 支持设置全局偏移量（Offset）。
   - 支持在任意位置插入 BPM 变更点、拍号变更点。
3. 在**歌词模式**下，输入或导入歌词文本，然后在已建立的网格上逐句/逐词打轴。
   - 打轴时若开启吸附，歌词时间会自动对齐到最近的拍子网格（精度可调整，如 1/4、1/8、1/16 音符等）。
   - 所有时间以**秒**（可含小数）为单位存储，吸附仅在 UI 层完成。

### 3.2 导入校对工作流

- 支持导入 TXT 纯文本歌词或 LRC / ASS 等已有时间轴的歌词，支持导入 JSON 格式工程文件。
- 导入后可在已有的时间轴上修改歌词文本（按行号匹配更新，保留原时间轴），或全部重打。
- 导入模块设计为可扩展架构。

### 3.3 双模式设计

- **时轴模式**：建立和编辑时间轴网格（BPM、拍号、Offset）。
- **歌词模式**：在网格上为歌词标记起止时间。
- 两种模式共享同一个播放器，节拍器和吸附功能全局可用。

### 3.4 歌词切词

- 同时支持**自动切词**与**手动切词**。
- 自动切词规则：
  - 中文：一字一词。
  - 英文：按空格分词。
  - 日语及其他语言暂不处理。
- 手动切词：
  - 参考 Aegisub，提供一个切词栏，用户可在词块中间点击切割，将句子从点击处分为两个词。
  - 切分结果支持撤销/重做。
- 工作流建议：
  - 可先按 `D` 逐词打轴/按 `Enter` 结束当前句子，待歌词断句标记完成后再统一切词。
- 批量切词：选中多句后统一切分。
- 切分后的词初始时间均为空（startTime/endTime 为 null），用户手动在波形/频谱上逐个标记。
- 词间允许保留间隙，由用户手动调整。

### 3.5 批量歌词编辑

- 以右侧固定面板提供文本编辑区。
- 基于行号匹配：
  - 新增行 → 创建无时间的新条目。
  - 删除行 → 删除对应条目（含其时间轴）。
  - 行号不变 → 更新文本，保留已有时间轴。
- 编辑操作可撤销/重做（Ctrl+Z / Ctrl+Y）。
- 处理已打轴或已切词的句子时，需考虑 diff 匹配，智能在句子前后增删时间条目。

### 3.6 导出

- 暂支持导出格式：LRC、ASS（SRT）、JSON（JSON 为本项目专有工程文件格式，可保存完整编辑状态）。
- 导出模块设计为可扩展架构。

## 4. 界面布局与设计

### 4.1 整体粗略布局

```
┌─ MenuBar（文件/编辑等菜单，点击打开菜单，非 hover）─┐
├─ TransportBar（播放控制、进度条、音量、模式切换）─┤
├─ MainView 默认高度 250px，可拖拽缩放 ─────────────┤
│  ├─ 波形（支持 autoCenter + autoScroll）          │
│  └─ 频谱（通过切换按钮显示）                      │
├─ WordTimelineBar（逐词条，三色词块）──────────────┤
├─ 模式面板（flex-1）───────────────────────────────┤
│  时轴模式: TimingPointsList（左） + TimingPanel（右）│
│  歌词模式: LyricsEditor（左） + 节拍器/吸附控制（右）│
└──────────────────────────────────────────────────┘
```

### 4.2 传输栏（TransportBar）

- 播放/暂停按钮。
- 快进/快退按钮（支持跳转到上下分隔线：`←` `→`；跳小节：`Shift+←` `Shift+→`）。
- 进度条（可拖拽）。
- 音量控制。
- 模式切换按钮（时轴模式/歌词模式）。

### 4.3 主视图区域（波形/频谱）

- 高度可由用户拖动缩放，默认 250px，不受其他元素影响。
- 波形与频谱通过切换显示，不同时可见。
- 频谱显示时支持调整频率缩放。
- 播放头与网格线同步滚动（随音频 currentTime 移动）。
- 网格刻度线根据当前吸附精度动态变化。
- 可显示 BPM/拍号变更标记。

### 4.4 逐词条（WordTimelineBar）

- 位于主视图正下方，类似 Aegisub 字幕编辑条。
- 以三色词块展示当前行所有词的状态：
  - 🟢 已打轴（可拖拽调整边界）
  - 🔴 当前目标词（等待打轴）
  - ⬛ 待打轴（尚未标记时间）
- 播放头指示当前播到的词位置。
- 跟随波形/频谱水平滚动。

### 4.5 时轴模式面板（TimingPanel）

右侧面板包含以下功能：

- **TAP BPM 圆盘按钮**
  - 最大保留最近 128 次敲击采样。
  - 连续三次敲击后开始计算 BPM。
  - 若两次敲击间隔超过 1 秒，自动清空采样。
- **BPM 微调按钮**：±0.1 / ±0.2 / ±0.5 / ±1
- **Offset 微调按钮**
  - 采用分区按钮设计：点击不同位置呈现不同偏移量选项（-10、-5、-2、-1、+1、+2、+5、+10 ms），按钮有明显的块状颜色渐变提醒。
- **拍号设置**：分子、分母完全自定义。
- **节拍器开关**：独立于播放器，可随时启停（追踪活跃音源以支持即时停止）。
- 吸附精度选择（1/2、1/4、1/8、1/16 等）。
- 左侧面板：**Timing Points 列表**
  - 以时间顺序显示所有 BPM 变更点和拍号变更点。
  - 支持添加、删除、编辑变更点。

### 4.6 歌词模式面板

- 左侧：歌词编辑器，支持选中、编辑、切词操作。
- 右侧：全局节拍器开关、吸附开关及精度选择。

## 5. 交互与快捷键

**系统应支持所有快捷键由用户自定义。**

### 5.1 全局快捷键

| 按键                       | 作用                          |
| -------------------------- | ----------------------------- |
| `Space`                    | 播放/暂停（全局，不区分模式） |
| `M`                        | 节拍器开关                    |
| `←` / `→`                  | 跳至上/下一个分隔线           |
| `Shift+←` / `Shift+→`      | 跳一小节                      |
| `Ctrl+Z`                   | 撤销                          |
| `Ctrl+Y` 或 `Ctrl+Shift+Z` | 重做                          |

### 5.2 时轴模式快捷键

| 按键 | 作用                    |
| ---- | ----------------------- |
| `B`  | TAP BPM（触发一次敲击） |

### 5.3 歌词模式快捷键

| 按键      | 作用                                                             |
| --------- | ---------------------------------------------------------------- |
| `D`       | 打轴并前进到下一个词（若当前词未标记起始时间则自动标记起始时间） |
| `Enter`   | 前进到下一行（如该句未标记结束时间则自动标记）                   |
| `Shift+D` | 打轴但不前进（仅标记当前词结束时间）                             |
| `Ctrl+D`  | 切词（对所选行执行切词）                                         |
| `Delete`  | 删除当前行（含时间轴）                                           |
| `C`       | 从当前句子开头播放，到句子结尾停止                               |
| `V`       | 从当前词开头播放，到词结尾停止                                   |

### 5.4 通用交互规则

- 全局快捷键在焦点位于需要文本输入的控件（如 input、textarea、select）时不触发；开关按钮、播放按钮等不拦截快捷键。
- 菜单栏由 hover 展开改为 click 展开，防止意外关闭。
- 所有可拖拽元素（如词块边界）提供实时反馈。

## 6. 非功能性需求

- 默认主题跟随系统 `prefers-color-scheme`，支持浅色与深色模式。
- 波形/频谱显示高度只能由用户手动拖动缩放，不能被其他界面元素影响。
- 时间轴滚动同步：波形、频谱、小节线、逐词条共同保持水平滚动位置一致。
- 节拍器需精确跟踪音频当前时间，关闭时立即停止所有发声，不能继续播放完当前小节。
- 编辑操作需支持完整的撤销/重做栈。

## 7. 设计参考

- osu! editor Timing 面板（TAP 测 BPM、Offset 设置、拍号变更交互）。
- Aegisub 的字幕时间轴编辑（拖拽调整、手动切词栏）。
